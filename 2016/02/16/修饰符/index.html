##类中的修饰符

一个Java源文件中至少包含一个使用public修饰的、类名与源文件名相同的类；

源文件中可以包含0个至多个没有public修饰的类，这些类的类名没有特殊的语法限制，可以使用任何合法的修饰符；

###类的可见性

##final类

###让类不能被继承
语法格式：

访问控制符 + final + class + 类名 + 类主体部分的内容

final修饰符很少被使用在类上，像我们之前所学的String类其实在源代码里面是被final修饰符所修饰的，所以我们不能写 XXX extends String

在某种程度上来说，用final修饰符修饰一个类和一个变量都代表他们“不可改变”； 一个是行为不能改变，一个是值不能改变。

##方法的修饰符
方法的访问修饰符更多种多样，有四个
分别为public, protected, 默认, private；

###public——没有限制的修饰符
使用public修饰符修饰的方法没有访问限制，对于本类、子类、同包的类以及不同包的类都是可见的；

###protected——仅对子类和同包的类可见
相比public，他不能对不同包的类可见，但是由于他对子类可见；
如果一个类，他是某protected类的子类，即使他们不同包，他们也是可见的，可以继承的。
只要符合其中一个条件即可。

###默认控制符——仅在本包中可见
因为默认控制符只能在本包中可见，如果有一个子类和父类不在同一个包中，那么我们可以认为，这个子类没有继承父类中那些使用默认访问符修饰的方法；

###private——仅对本类可见

###final——让方法不能被覆盖
使用final关键字修饰的方法不能被子类覆盖
final关键字一定要放在方法的访问控制符之后。

##静态方法
使用static修饰的方法称作静态方法，又称为类方法。同样的，使用static修饰的变量称作类变量或者静态变量。

语法如下: 访问控制符+static+返回值类型+方法名字+方法参数+方法体；

static和final都要放在访问控制符和返回值类型之间，但是如果两个都有的时候没有顺序要求。

静态方法内部的代码只能使用本方法的局部变量、方法的参数、静态变量和静态方法。不能使用非静态方法和变量。
